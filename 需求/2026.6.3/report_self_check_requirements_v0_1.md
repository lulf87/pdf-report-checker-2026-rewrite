## 0. 本文件目的

本文件用于重新定义“报告自身核对”模块的产品需求、业务范围、规则口径、输入输出、结果状态和待确认事项。

本文件不涉及代码设计，不涉及技术栈选型，不涉及数据库/API 具体实现细节。确认本文件后，下一阶段再进入技术架构设计。

当前项目包含两个业务模块：

1. PTR 条款核对：检验报告与产品技术要求之间的条款一致性核对。
2. 报告自身核对：单份检验报告内部字段、标签、照片、检验项目和页码等规则核对。

本文件建议 v1 仅聚焦“报告自身核对 C01-C11”。PTR 条款核对暂不进入 v1 范围，避免新项目一开始再次混乱。

## 2. 产品目标

### 2.1 业务目标

系统用于对固定格式医疗器械检验报告进行自动化预审，帮助人工审查人员发现报告内部不一致、漏填、结论逻辑错误、照片/标签缺失、页码错误等问题。

系统不是最终审批系统，不替代人工签发结论。系统输出应作为“自动核对结果”和“人工复核依据”。

### 2.2 用户目标

目标用户主要包括：

| 用户角色 | 主要诉求 |
|---|---|
| 报告编制人员 | 上传报告后快速发现格式、字段、结论和页码问题 |
| 报告审核人员 | 根据系统列出的失败项和证据快速复核 |
| 管理人员 | 统计常见错误、提高报告出具一致性 |
| 系统维护人员 | 调整规则、更新样本、定位解析失败原因 |

### 2.3 成功标准

v1 成功标准建议如下：

| 类别 | 标准 |
|---|---|
| 功能覆盖 | 覆盖 C01-C11 规则 |
| 可解释性 | 每条 FAIL/REVIEW 结果必须有证据 |
| 稳定性 | 对金标准样本集结果稳定，不因小改动导致规则漂移 |
| 可回归 | 每条规则至少有正常样本和异常样本 |
| 可人工复核 | 低置信度、无法定位、OCR 不可靠时，不直接误判为报告错误 |
| 可导出 | 支持导出 JSON 和 PDF 核对报告 |

## 3. v1 范围定义

### 3.1 v1 包含范围

v1 包含以下内容：

1. 单份检验报告 PDF 上传。
2. 报告结构解析：页面、文本、表格、页码、照片页、标签说明文字。
3. 报告首页、第三页、样品描述表、照片页、检验项目表格的字段抽取。
4. OCR 识别中文标签内容。
5. 执行 C01-C11 报告自身核对规则。
6. 返回结构化 JSON 结果。
7. 前端展示核对摘要和规则明细。
8. 导出 PDF 核对报告。
9. 对失败项、需复核项提供证据。

### 3.2 v1 不包含范围

v1 暂不包含以下内容：

| 不包含项 | 说明 |
|---|---|
| PTR 条款核对 | 作为后续 v2 模块独立设计 |
| DOCX 原生解析 | 如确需支持，应先转换为 PDF，再进入同一处理流程 |
| 报告模板编辑器 | v1 只处理固定格式，不提供模板配置 UI |
| 用户登录与权限 | v1 可先假设内网可信环境 |
| 多租户隔离 | 正式商用前再设计 |
| 审批流 | 系统只输出核对结果，不做审批动作 |
| LLM 直接判定规则通过/失败 | 规则判断必须由确定性程序完成 |
| 自动修改 PDF | 系统只核对和导出结果，不修改原始报告 |
| 批量任务 | v1 可暂缓，v1.1 再支持 |
| Excel 导出 | 当前列为 v1.1 候选能力 |

### 3.3 后续版本建议

| 版本 | 建议范围 |
|---|---|
| v1 | 报告自身核对 C01-C11，PDF 输入，JSON/PDF 输出 |
| v1.1 | Excel 导出、批量上传、规则开关、样本回归工具 |
| v1.5 | OCR/VLM 增强、复杂扫描件处理、证据截图 |
| v2 | PTR 条款核对，包括条款层级、表格引用、差异高亮 |
| v3 | 桌面打包、企业部署、权限、审计日志 |

## 4. 输入规格

### 4.1 输入文件

| 项目 | 规格 |
|---|---|
| 文件类型 | `.pdf` |
| 文件内容 | 固定格式检验报告 |
| 最大文件大小 | 50MB |
| 最大页数 | 建议 200 页 |
| 是否允许扫描件 | 允许，但结果可能产生 REVIEW |
| 是否允许加密 PDF | v1 不支持 |
| 是否允许损坏 PDF | 不支持，返回 SYSTEM_ERROR |

### 4.2 输入报告的结构假设

系统默认报告包含以下结构：

| 页面/区域 | 识别方式 | 用途 |
|---|---|---|
| 首页 | PDF 第 1 页 | 提取委托方、样品名称、型号规格 |
| 第三页/检验报告首页 | 页眉或页面文本包含“检验报告首页” | 提取第三页字段，参与 C01-C03 |
| 样品描述页 | 页面文本包含“样品描述” | 提取部件名称、规格型号、序列号/批号、生产日期、失效日期、备注 |
| 照片页 | 页面文本包含“检验报告照片页”，或包含照片/标签相关说明 | 提取照片 caption、中文标签 caption、标签 OCR 信息 |
| 检验项目表格 | 表头包含序号、检验项目、标准条款、标准要求、检验结果、单项结论、备注 | 执行 C07-C10 |
| 页码区域 | 从第三页开始，右上角 `共XXX页 第Y页` | 执行 C11 |

### 4.3 不支持或需复核的输入情况

| 情况 | 建议处理 |
|---|---|
| 找不到“检验报告首页” | C01-C03 标记 REVIEW |
| 找不到“样品描述”表格 | C04-C06 标记 REVIEW |
| 找不到检验项目表格 | C07-C10 标记 REVIEW |
| 找不到页码文本 | C11 标记 REVIEW |
| OCR 无结果或置信度过低 | 相关规则标记 REVIEW |
| PDF 文本顺序明显错乱 | 相关规则标记 REVIEW |
| 页数超过限制 | 上传失败，返回 SYSTEM_ERROR |
| 文件超过大小限制 | 上传失败，返回 SYSTEM_ERROR |

## 5. 输出规格

### 5.1 输出形态

| 输出 | v1 是否支持 | 说明 |
|---|---|---|
| Web 可视化结果 | 是 | 展示摘要、规则状态、失败明细和证据 |
| JSON 结果 | 是 | 作为系统核心结果格式，便于回归测试和集成 |
| PDF 核对报告 | 是 | 供人工留档和流转 |
| Excel 表格 | 暂缓 | 建议 v1.1 支持 |

### 5.2 任务状态

| 状态 | 含义 |
|---|---|
| PENDING | 已创建任务，等待处理 |
| PROCESSING | 正在解析/抽取/核对 |
| COMPLETED | 处理完成，可查看结果 |
| SYSTEM_ERROR | 系统处理异常，未能生成完整结果 |

### 5.3 规则结果状态

规则结果状态用于描述某条规则的判定结果。

| 状态 | 含义 | 是否代表报告错误 |
|---|---|---|
| PASS | 核对通过 | 否 |
| FAIL | 核对不通过，报告存在明确问题 | 是 |
| REVIEW | 系统无法可靠判断，需要人工复核 | 不一定 |
| SKIP | 规则不适用 | 否 |
| SYSTEM_ERROR | 规则执行异常 | 不一定，属于系统问题 |

> 注意：旧项目中存在 `pass/error/warning/skipped` 等状态。v2 建议区分“规则状态”和“严重等级”，避免把 OCR 不确定性误判为报告错误。

### 5.4 严重等级

严重等级用于辅助排序和展示。

| 等级 | 含义 | 建议处理 |
|---|---|---|
| ERROR | 明确错误，必须修复 | 报告编制人员修正 |
| WARN | 需人工复核或低置信度 | 审核人员判断 |
| INFO | 信息展示 | 参考 |

### 5.5 每条结果必须包含的信息

每条 FAIL 或 REVIEW 结果至少包含：

| 字段 | 必须性 | 说明 |
|---|---|---|
| rule_id | 必须 | 如 C01 |
| rule_name | 必须 | 规则名称 |
| status | 必须 | PASS/FAIL/REVIEW/SKIP/SYSTEM_ERROR |
| severity | 必须 | ERROR/WARN/INFO |
| message | 必须 | 面向用户的中文说明 |
| finding_code | 建议 | 机器可读错误代码 |
| page_number | 必须 | 证据所在页码 |
| field_name | 视规则而定 | 字段名 |
| expected | 视规则而定 | 期望值 |
| actual | 视规则而定 | 实际值 |
| raw_text | 必须 | 取到的原始文本 |
| normalized_text | 建议 | 归一化后的比较值 |
| source | 必须 | text/table/ocr/vlm/manual |
| confidence | 视来源而定 | OCR/VLM 置信度 |
| bbox | 建议 | PDF 坐标，用于后续证据截图 |

## 6. 通用业务约定

### 6.1 原始值与归一化值

所有抽取字段应同时保留：

| 值类型 | 用途 |
|---|---|
| raw_value | 原文展示、证据追溯 |
| normalized_value | 规则比较 |

不允许为了比较方便而覆盖原始值。

### 6.2 严格一致原则

默认情况下，字段比对采用严格一致：

1. 大小写敏感。
2. 全半角敏感。
3. 空格敏感。
4. 标点敏感。
5. 字符顺序敏感。

但存在明确声明的业务等价规则，见下文。

### 6.3 等价规则

以下值在部分规则中视为等价：

| 值 | 等价值 |
|---|---|
| 空字符串 | `/` |
| `/` | 空字符串 |
| `见实物` | 空字符串或 `/`，仅限声明适用的字段 |

> 待确认：`见实物` 是否应全局等价，还是只适用于样品描述表与标签字段。

### 6.4 “见\"样品描述\"栏”规则

第三页扩展字段中可能出现 `见"样品描述"栏`。

建议将它视为特殊业务值，不参与普通字段 OCR 比对。

适用字段：

1. 型号规格。
2. 生产日期。
3. 产品编号/批号。

判定原则：

| 情况 | 判定 |
|---|---|
| 三个字段全部为 `见"样品描述"栏` | PASS |
| 三个字段全部不是 `见"样品描述"栏` | 进入标签字段比对 |
| 只有部分字段为 `见"样品描述"栏` | FAIL |

### 6.5 OCR 与 LLM/VLM 使用原则

OCR/VLM 只能作为字段抽取来源，不得直接给出规则是否通过的最终结论。

| 类型 | 用途 | 是否允许最终判定 |
|---|---|---|
| PDF 文本解析 | 主来源 | 否，仍由规则引擎判断 |
| PDF 表格解析 | 主来源 | 否，仍由规则引擎判断 |
| OCR | 扫描件/标签兜底 | 否 |
| VLM/LLM | OCR 失败或低置信度时辅助抽取 | 否 |
| 规则引擎 | 最终判断 | 是 |

### 6.6 解析失败与报告错误分离

解析失败不等于报告错误。

| 场景 | 错误类型 |
|---|---|
| 字段定位失败 | REVIEW |
| 表格无法解析 | REVIEW |
| OCR 低置信度 | REVIEW |
| 程序异常 | SYSTEM_ERROR |
| 明确字段不一致 | FAIL |
| 明确漏填 | FAIL |
| 明确逻辑错误 | FAIL |

## 7. 核对规则总览

| 编号 | 规则名称 | 核对对象 | 默认状态等级 | v1 优先级 |
|---|---|---|---|---|
| C01 | 首页与第三页一致性 | 委托方、样品名称、型号规格 | ERROR | P1 |
| C02 | 第三页扩展字段 | 型号规格、生产日期、产品编号/批号 | ERROR | P2 |
| C03 | 生产日期格式 | 第三页表格与标签生产日期格式 | ERROR | P2 |
| C04 | 样品描述表格 | 部件字段与中文标签字段 | ERROR/WARN | P3 |
| C05 | 照片覆盖性 | 每个部件至少一张照片 | ERROR | P3 |
| C06 | 中文标签覆盖 | 每个部件至少一张中文标签 | ERROR | P3 |
| C07 | 检验项目-单项结论 | 检验结果与单项结论逻辑 | ERROR | P1 |
| C08 | 检验项目-非空字段 | 检验结果、单项结论、备注 | ERROR | P0 |
| C09 | 检验项目-序号连续性 | 序号列 | ERROR | P0 |
| C10 | 检验项目-续表标记 | 跨页续表 | ERROR | P1 |
| C11 | 页码连续性 | `共XXX页 第Y页` | ERROR | P0 |

优先级解释：

| 优先级 | 含义 |
|---|---|
| P0 | 第一批实现，确定性强，适合打地基 |
| P1 | 第二批实现，依赖表格和字段抽取 |
| P2 | 第三批实现，依赖标签 OCR |
| P3 | 第四批实现，依赖照片/caption/OCR/部件匹配，复杂度最高 |

## 8. 页面与结构定位规则

### 8.1 首页定位

| 项目 | 规则 |
|---|---|
| 页面 | PDF 第 1 页 |
| 主要字段 | 委托方、样品名称、型号规格 |
| 抽取方式 | 优先表格单元格定位；文本正则作为兜底 |
| 失败处理 | 找不到字段时，C01 对应字段 REVIEW |

### 8.2 第三页/检验报告首页定位

| 项目 | 规则 |
|---|---|
| 页面识别 | 页面文本去空白后包含“检验报告首页” |
| 主要字段 | 委托方、样品名称、型号规格、检验项目、生产日期、产品编号/批号、委托方地址 |
| 抽取方式 | 优先表格结构；文本正则作为兜底 |
| 失败处理 | 找不到页面时，C01-C03 REVIEW |

> 待确认：业务上称“第三页”，但实际定位是否应完全依赖页码顺序，还是依赖“检验报告首页”锚点。建议以锚点为主，页码顺序为辅助。

### 8.3 样品描述页定位

| 项目 | 规则 |
|---|---|
| 页面识别 | 页面文本包含“样品描述” |
| 表格识别 | 表格包含部件名称/产品名称/名称等列 |
| 字段 | 部件名称、规格型号、序列号/批号、生产日期、失效日期、备注 |
| 忽略列 | 序号、备注不参与标签字段一致性比对，但备注参与“本次检测未使用”判断 |

### 8.4 照片页定位

| 项目 | 规则 |
|---|---|
| 页面识别 | 页面文本包含“检验报告照片页”或包含图/照片/标签/Photo/Plate 等关键词 |
| 抽取对象 | 图片说明文字 caption、中文标签说明、标签 OCR 文本 |
| 风险 | 仅用页面文本关键词可能误判，后续应加入版面/图片区域识别 |

### 8.5 检验项目表格定位

表头需包含以下列名：

1. 序号。
2. 检验项目。
3. 标准条款。
4. 标准要求。
5. 检验结果。
6. 单项结论。
7. 备注。

若缺少任一核心列，应标记 C07-C10 为 REVIEW。

### 8.6 页码定位

| 项目 | 规则 |
|---|---|
| 起始范围 | 从“检验报告首页”所在页开始，通常对应第三页 |
| 页码格式 | `共XXX页 第Y页` |
| 位置 | 页面右上角，文本定位优先 |
| 失败处理 | 找不到页码时 C11 REVIEW |

## 9. 详细核对规则台账

### C01 首页与第三页一致性

## 10. 照片 caption 与主体名规则

### 10.1 Caption 定义

Caption 是照片或标签下方/附近的说明文字，用于识别照片或标签对应的部件。

示例：

1. `№113导管动态压力检测仪`
2. `№113导管动态压力检测仪 中文标签`
3. `25: 心脏脉冲电场消融仪-主机 中文标签样张`

### 10.2 主体名提取规则

主体名提取步骤建议如下：

1. 去除编号前缀，如 `№113`、`No.113`、`25:`。
2. 去除方位词，如前侧、后侧、左侧、右侧、顶面、底面等。
3. 去除类别词，如中文标签样张、中文标签、标签、照片等。
4. 清理所有空白字符。
5. 得到主体名后用于 C05/C06/C04 匹配。

### 10.3 主体名匹配规则

| 优先级 | 方式 | 示例 |
|---|---|---|
| 1 | 精确匹配 | 部件名称 = 主体名 |
| 2 | 部件名称包含于主体名 | `主机` 匹配 `主机及推车` |
| 3 | 主体名包含于部件名称 | `触摸屏连接线缆` 匹配 `心脏脉冲电场消融仪-触摸屏连接线缆（30m）` |
| 4 | 人工复核 | 多个候选无法唯一匹配 |

待确认事项：

1. 部分匹配的边界字符集合是否固定。
2. 是否需要维护产品名称别名表。
3. 如果一个 caption 匹配多个部件，是否全部通过，还是进入 REVIEW。

## 11. 特殊业务规则

### 11.1 “本次检测未使用”部件

当样品描述表备注列包含 `本次检测未使用` 时：

| 场景 | 建议结果 |
|---|---|
| 无照片 | PASS 或 SKIP |
| 无中文标签 | PASS 或 SKIP |
| 有照片/标签且字段一致 | PASS |
| 有照片/标签但字段不一致 | WARN 或 REVIEW |
| 表格自身必填字段为空 | 仍按 C04/C08 规则处理，待确认 |

建议：

1. C05/C06 对未使用部件无照片/标签不报错。
2. C04 若未使用部件存在标签且字段不一致，不直接 FAIL，标记 WARN/REVIEW。
3. 输出中明确说明“不报错原因：备注为本次检测未使用”。

### 11.2 同名多行部件

当样品描述表出现同名多行部件时，应使用非空字段联合键区分。

联合键候选字段：

1. 部件名称。
2. 规格型号。
3. 序列号/批号。
4. 生产日期。
5. 失效日期。

待确认：如果联合键仍无法区分，是 REVIEW 还是 FAIL。建议 REVIEW。

### 11.3 空白、斜杠、见实物

| 表格值 | 标签值 | 建议判定 |
|---|---|---|
| 空白 | 无字段 | PASS |
| `/` | 无字段 | PASS |
| `见实物` | 无字段 | PASS，限样品描述字段 |
| 空白或 `/` | 有字段值 | FAIL，表格漏填 |
| 有字段值 | 无字段 | FAIL 或 REVIEW，取决于 OCR 可靠性 |
| 有字段值 | 不同字段值 | FAIL |

## 14. 前端展示需求草案

v1 前端只做审查台，不追求复杂视觉效果。

### 14.1 页面

| 页面 | 功能 |
|---|---|
| 上传页 | 上传 PDF，选择是否启用 OCR/VLM 增强 |
| 任务页 | 查看任务状态、进度、错误信息 |
| 结果总览页 | 展示 PASS/FAIL/REVIEW/SKIP 统计 |

### 14.2 结果筛选

结果页应支持：

1. 查看全部。
2. 仅查看 FAIL。
3. 仅查看 REVIEW。
4. 按规则编号筛选。
5. 按严重等级筛选。
6. 搜索字段名或部件名。

### 14.3 证据展示

每条失败项应展示：

1. 错误说明。
2. 规则编号。

4. 期望值。
5. 实际值。
6. 页码。
7. 原始文本。
8. OCR 置信度，如有。
9. 后续如支持 bbox，可显示高亮截图。
